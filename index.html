Part 1: URL & Link（≤ 30 分鐘）

🎮 遊戲 1: Wiki 連結挑戰賽

互動式遊戲體驗

點擊次數: 0
你的路徑：

遊戲規則

  • 從指定的維基百科條目 A，只透過點擊條目內的連結，最終到達另一個指定條目 B
  • 點擊次數越少越好！
  • 不能用瀏覽器的「上一頁」功能，不能直接搜尋

💡 這個遊戲會讓學生自己發現：連結不是隨機的，而是形成了某種「網絡」！

🔗 什麼是「連結」？

剛剛的遊戲好玩嗎？你們有沒有注意到，每次點擊那個藍色的字，你就會「跳」到另一個完全不同的頁面？這個「跳躍」，就是我們今天要研究的第一個主角：連結（Link）！

想像一下，網際網路就像一個超級巨大的圖書館，而每個網頁就是一本書。連結就像是書裡面寫著「如果你想知道更多，請翻到第 X 頁」或「請參考另一本書《XXX》」的標註。

但這個「圖書館」有個超酷的功能：你不需要真的走到另一個書架，也不需要翻頁——只要點一下，瞬間就到了！這就是數位世界的魔法。

🧩 URL 的秘密結構

一個完整的 URL（Uniform Resource Locator）——也就是「網址」，就像是網路世界的門牌地址！

互動式 URL 拆解

[1] 協定 Protocol
https://

規定「怎麼傳資料」，就像郵局規定你要用「平信」還是「掛號」

[2] 域名 Domain
www.wikipedia.org

「誰」在提供這個網頁，就像你家的門牌號碼

[3] 路徑 Path
/wiki/Graph_theory

你要的「具體是哪個東西」，就像房間號碼

Part 2: Graph 圖論（≤ 40 分鐘）

🎮 遊戲 2: 一筆畫挑戰賽

互動式一筆畫遊戲

點擊頂點開始畫圖！每條邊只能經過一次，最後回到起點。

一筆畫的秘密（歐拉定理）

一個圖可以一筆畫完（回到起點）的充要條件是：所有頂點的分支度都是偶數！

為什麼？想像你走進一個點，就要走出來——進一條、出一條，所以線的數量必須是成對的，也就是偶數！

🌐 什麼是「圖」？

你們剛才玩的那些「點點連連」的東西，在數學裡有個專門的名字，叫做「圖（Graph）」！

⚠️ 注意！這裡的「圖」不是指照片或圖片，而是數學家用來描述「連結關係」的一種工具！

一個圖（Graph）由兩個部分組成：

  • 頂點（Vertex / Node） → 就是那些「點」
  • 邊（Edge） → 就是連接點的「線」

生活中的「圖」例子

👥
社群網路

頂點 = 每個人
邊 = 好友關係

🚇
交通網路

頂點 = 車站
邊 = 路線

🌐
網際網路

頂點 = 網頁
邊 = 超連結

🏗️ 最小生成樹（MST）

想像你是一個網路公司的工程師，你要在幾個城市之間鋪設光纖電纜。但是鋪電纜很貴，你的老闆說：「要花最少的錢，但要讓所有城市都能互相通訊！」

這時候，你就需要找出一個「最小生成樹」！

Part 3: Merge [URL + Graph]（≤ 20 分鐘）

🎭 揭曉時刻：一切都是圖！

還記得一開始我們玩的「維基百科連結遊戲」嗎？那個遊戲其實就是在一個「圖」上面走路！

關鍵發現

網際網路 = 一個巨大的「有向圖」（Directed Graph）

  • 頂點（Vertex） = 每一個網頁（URL）
  • 邊（Edge） = 網頁之間的超連結（Link）

如果網頁 A 有一個連結指向網頁 B，不代表 B 也有連結指向 A。就像單行道——你可以從 A 走到 B，但不一定能從 B 走回 A！

為什麼這個概念這麼重要？

1. Google 的 PageRank 演算法

Google 會計算「有多少網頁指向某個網頁」，來判斷這個網頁的「重要性」。越多網頁指向你 = 你越重要 = 搜尋排名越前面！

2. 推薦系統（YouTube、Netflix）

「看過影片 A 的人，也看了影片 B、C、D……」這些連結關係也是一個「圖」！

3. 社群網路分析

「你可能認識的人」= 你的朋友的朋友（圖的鄰居的鄰居）

🎉 總結：今天學到了什麼？

  • ✓ URL = 協定 + 域名 + 路徑，是網路世界的門牌地址
  • ✓ 連結（Link） 讓我們在網頁之間跳躍
  • ✓ 圖（Graph） = 頂點 + 邊，用來描述連結關係
  • ✓ 樹（Tree） = 沒有迴圈的圖
  • ✓ 最小生成樹（MST）：用最少成本連接所有頂點
  • ✓ 網際網路 = 有向圖，網頁是頂點，連結是有方向的邊
